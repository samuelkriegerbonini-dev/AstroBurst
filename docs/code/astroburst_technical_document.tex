    \section{Executive Summary}
%% ============================================================================

    AstroBurst is a cross-platform desktop application for processing, analyzing, and visualizing astronomical FITS and ASDF images. It is the first astronomical image processor built on the \textbf{Rust/Tauri/WebGPU} stack, combining native-level performance with a modern, accessible user interface.

    The application targets two user segments:
    \begin{itemize}[nosep]
    \item Professional astronomers working with observatory data (JWST, Hubble, Roman Space Telescope, ground-based IFU instruments)
    \item Advanced astrophotographers processing deep-sky narrowband and broadband imaging data
    \end{itemize}

    Version 0.4.6 delivers a complete processing pipeline from raw FITS/ASDF ingestion through calibration, stacking, alignment, color composition, and export, with \textbf{47 backend commands} exposed through a typed TypeScript API layer and an 11-step ComposeWizard pipeline.

    \subsection{Backend (Rust)}

    The Rust backend is organized into six layers:

    \begin{description}[style=nextline, leftmargin=2cm]
    \item[\texttt{cmd/}] Tauri command handlers organized by domain (47 registered commands). Each module receives deserialized JSON arguments, delegates to core/domain logic, and returns JSON or binary responses.
    \item[\texttt{core/}] Pure computational modules: alignment (phase correlation, affine with triangle asterism matching), analysis (histogram, FFT, stars), astrometry (plate solve, WCS, SPCC), compose (RGB, channel blend, SCNR, white balance, LRGB), cube (spectrum, frames), imaging (STF, deconvolution, wavelet, background, masked stretch, PSF estimation, star mask), metadata (header discovery, filter detection), stacking (sigma-clip, drizzle), synth (star field, PSF, noise).
    \item[\texttt{domain/}] Orchestration layer coordinating core modules for complex multi-step operations (calibration masters, drizzle-from-paths, cube processing, plate solve).
    \item[\texttt{infra/}] Infrastructure: Arc-based zero-copy cache with ORIG/KEY dual-layer for non-destructive composite editing, config management, FITS I/O (mmap parser, MEF scanner, FITS writer with multi-BITPIX), ASDF reader (YAML tree parser, binary block extractor, zlib/bzip2/lz4 decompressor, Roman data model converter, gWCS bridge), binary IPC encoding, progress tracking, PNG rendering (8/16-bit grayscale and RGB).
    \item[\texttt{types/}] Shared type definitions: image stats, STF params, stacking/drizzle/RL configs, compose types (AlignMethod, WhiteBalance, BlendWeight), SCNR, error types, FITS header abstraction, 310+ constants.
    \item[\texttt{math/}] Mathematical primitives: NaN-safe median/MAD, sigma clipping, SIMD log approximation (Cephes degree-8 minimax, $<1$\,ULP precision).
    \end{description}

    \subsection{Non-Destructive Composite Pipeline (v0.4.5)}
    \label{sec:nondestructive}

    The composite cache uses a dual-layer architecture:

    \begin{infobox}[ORIG/KEY Cache Architecture]
        \begin{center}
            \begin{tabular}{lll}
                \toprule
                \textbf{Layer} & \textbf{Keys} & \textbf{Semantics} \\
                \midrule
                ORIG (immutable) & \code{\_\_composite\_orig\_r/g/b} & Written once by blend/compose/RGB-FITS load \\
                KEY (working) & \code{\_\_composite\_r/g/b} & Reconstructed: ORIG $\times$ WB $\to$ KEY, or ORIG $\times$ WB $\to$ SCNR $\to$ KEY \\
                \bottomrule
            \end{tabular}
        \end{center}
        Every downstream operation (\code{calibrate\_composite\_cmd}, \code{apply\_scnr\_cmd}) reads from ORIG, applies the full transform chain, and writes to KEY. This makes WB and SCNR idempotent: applying the same factors twice produces identical results. \code{reset\_wb\_cmd} copies ORIG $\to$ KEY.
    \end{infobox}

    \subsection{Frontend (React + TypeScript)}

    The frontend is organized around the ComposeWizard (11-step pipeline) and split preview contexts:

    \begin{description}[style=nextline, leftmargin=2cm]
    \item[\texttt{services/}] 12 domain-specific service modules wrapping Tauri IPC: compose, fits, analysis, header, cube, stacking, processing, config, astrometry, export, synth, tiles.
    \item[\texttt{ComposeWizard}] 11-step pipeline: Channels $\to$ Stack $\to$ Align $\to$ Crop $\to$ BG $\to$ Blend $\to$ Color $\to$ Mask $\to$ Stretch $\to$ Adjust $\to$ Export. Internal \code{useReducer} state with step completion tracking and suggested-next-step logic.
    \item[Preview Contexts] 8 split React contexts (FileCtx, DoneFilesCtx, HistCtx, CubeCtx, RgbCtx, RenderCtx, RawPixelsCtx, NarrowbandCtx) for granular re-render control.
    \item[GPU Rendering] WebGPU singleton with WGSL fragment shader for real-time STF. CPU worker fallback with identical MTF implementation. Both use symmetric denominator protection and padding threshold matching the Rust backend.
    \end{description}

    \subsection{Screen Transfer Function (STF)}
    \label{sec:stf}

    The STF implements the PixInsight-style \textbf{Midtone Transfer Function} (MTF). Given a normalized pixel value $x \in [0, 1]$ and midtone parameter $m \in (0, 1)$:

    \begin{equation}
        \boxed{
            \text{MTF}(x, m) = \frac{(m - 1) \cdot x}{(2m - 1) \cdot x - m}
        }
        \label{eq:mtf}
    \end{equation}

    \textbf{Properties:}
    \begin{itemize}[nosep]
    \item $\text{MTF}(0, m) = 0$ and $\text{MTF}(1, m) = 1$ for all $m$
    \item $\text{MTF}(m, m) = 0.5$ --- the midtone maps to 50\% brightness
    \item $m < 0.5$ brightens the image; $m > 0.5$ darkens it
    \item When $m = 0.5$: $\text{MTF}(x, 0.5) = x$ (identity)
    \end{itemize}

    \begin{notebox}[Denominator Protection (v0.4.5)]
        The denominator $D = (2m - 1)x - m$ changes sign across the valid domain. The correct guard is:
        \[
            \text{if } |D| < \varepsilon \text{ then return } x; \quad \text{if } |m - 0.5| < \varepsilon \text{ then return } x
        \]
        The v0.4.2 GPU shader incorrectly used $\max(D, \varepsilon)$ which forces $D > 0$, inverting the transfer function for $m < 0.5$. v0.4.5 uses symmetric $|D|$ protection across GPU (WGSL), CPU worker (JS), and Rust backend, achieving pixel-identical output.
    \end{notebox}

    \textbf{Padding pixels} ($v \leq 10^{-7}$) are rendered as black across all three rendering paths, matching the Rust \code{is\_valid\_pixel} threshold.

    The \textbf{auto-stretch} computes parameters from image statistics:

    \begin{align}
        \text{shadow} &= \max\left(0,\; \tilde{x}_{\text{norm}} + k \cdot \sigma_{\text{norm}}\right), \quad k = -2.8 \label{eq:shadow} \\[6pt]
        \text{midtone} &= \text{MTF}_{\text{balance}}^{-1}\left(\frac{\tilde{x}_{\text{norm}} - \text{shadow}}{1 - \text{shadow}},\; t_{\text{bg}}\right), \quad t_{\text{bg}} = 0.25 \label{eq:midtone}
    \end{align}

    where $\tilde{x}_{\text{norm}}$ and $\sigma_{\text{norm}}$ are the normalized median and sigma (MAD $\times$ 1.4826).

    \subsection{Asinh Stretch}

    \begin{equation}
        \boxed{
            f(x) = \frac{\operatorname{asinh}(\alpha \cdot x)}{\operatorname{asinh}(\alpha)}
        }
        \label{eq:asinh}
    \end{equation}

    Maps $[0, 1] \to [0, 1]$, approximately linear for small $x$ (preserving faint detail), logarithmic compression for bright values. $\alpha \in [1, 500]$ controls aggressiveness.

    \subsection{Masked Stretch (v0.4.2)}
    \label{sec:masked_stretch}

    Iterative MTF stretch with star mask protection. At each iteration $k$:

    \begin{align}
        M^{(k)} &= \text{StarMask}(I^{(k)},\; \text{growth},\; \text{softness}) \\
        I^{(k+1)} &= (1 - \alpha \cdot M^{(k)}) \cdot \text{MTF}(I^{(k)}) + \alpha \cdot M^{(k)} \cdot I^{(k)}
    \end{align}

    where $\alpha$ is the protection amount (default 0.85) and growth controls the mask dilation radius. Convergence when background level reaches target (default 0.25). JWST-optimized defaults: growth 0.8, protection 96\%.

    \subsection{Cross-Correlation Alignment (Phase Correlation)}
    \label{sec:alignment_phase}

    \begin{equation}
        \text{shift} = \arg\max_{(\Delta x, \Delta y)} \mathcal{F}^{-1}\!\left\{\frac{\hat{R} \cdot \hat{T}^*}{|\hat{R} \cdot \hat{T}^*|}\right\}
        \label{eq:phase_corr}
    \end{equation}

    Hann window: $w(n) = 0.5(1 - \cos(2\pi n / N))$. Confidence: $z = (P_{\text{peak}} - \mu_P) / \sigma_P$. Coarse-to-fine: downsample to $512 \times 512$, refine on centered crop.

    \subsection{Star-Based Affine Alignment (v0.4.0)}
    \label{sec:alignment_affine}

    Triangle descriptor (scale-invariant):
    \begin{equation}
        T = \left(\frac{d_{\text{mid}}}{d_{\text{short}}},\; \frac{d_{\text{long}}}{d_{\text{short}}}\right), \quad \text{match if } |T_r - T_t| < 0.005
    \end{equation}

    RANSAC (500 iterations) fits 6-DOF affine or 4-DOF rigid:
    \begin{equation}
        \begin{pmatrix} x_t \\ y_t \end{pmatrix} =
        \begin{pmatrix} a & b \\ c & d \end{pmatrix}
        \begin{pmatrix} x_r \\ y_r \end{pmatrix} +
        \begin{pmatrix} t_x \\ t_y \end{pmatrix}
    \end{equation}

    Sanity: offset $< 25\%$, rotation $< 5^{\circ}$, scale $\in [0.85, 1.15]$, residual $< 3$\,px, inlier ratio $> 30\%$. Fallback: affine $\to$ rigid $\to$ phase correlation $\to$ identity. Warp via bicubic Catmull-Rom, parallelized per row.

    \subsection{Sigma-Clipped Stacking}
    \label{sec:sigma_stack}

    \begin{algorithm}[H]
\SetAlgoLined
\Input{Frames $\{F_1, \ldots, F_N\}$, $\sigma_{\text{low}}, \sigma_{\text{high}}$, max iters $K$}
\Output{Stacked image $S$}
\ForEach{pixel $(i, j)$}{
    $V \leftarrow \{F_k(i,j)\}$, iter 0: $\mu = \text{median}(V)$, $\sigma = \text{MAD}(V) \times 1.4826$\;
    iters $1{+}$: $\mu = \text{mean}(V)$, $\sigma = \text{std}(V)$\;
    reject $v \notin [\mu - \sigma_{\text{low}} \cdot \sigma,\; \mu + \sigma_{\text{high}} \cdot \sigma]$\;
    $S(i,j) \leftarrow \text{mean}(V)$\;
}
\caption{Sigma-Clipped Stacking (median+MAD first iter, mean+std subsequent)}
    \end{algorithm}

    \subsection{Channel Blending (v0.4.2)}
    \label{sec:blend}

    Given $N$ input channels $C_k$ and weight matrix $W$:

    \begin{equation}
        \text{Out}_c = \sum_{k=1}^{N} W_{k,c} \cdot C_k, \quad c \in \{R, G, B\}
    \end{equation}

    Preset weights (e.g., SHO: $\text{SII} \to R$, $\text{H}\alpha \to G$, $\text{OIII} \to B$) use named channelId matching with \textbf{positional fallback} (v0.4.5): when preset channelIds don't match filled bins, weights map positionally to the first $N$ filled bins.

    \subsection{White Balance (v0.4.0, non-destructive v0.4.5)}
    \label{sec:white_balance}

    Reference channel selection by minimum coefficient of variation:

    \begin{equation}
        k^* = \arg\min_{k \in \{R, G, B\}} \frac{\text{MAD}_k}{\tilde{C}_k}; \qquad
        w_j = \frac{\tilde{C}_{k^*}}{\tilde{C}_j},\; j \neq k^*; \quad w_{k^*} = 1
    \end{equation}

    In v0.4.5, WB is applied from ORIG cache: $\text{KEY}_c = \text{ORIG}_c \times w_c$. Idempotent.

    \subsection{SCNR (v0.4.0, non-destructive v0.4.5)}
    \label{sec:scnr}

    Two limit methods and optional luminance preservation:

    \begin{align}
        G' &= \min\!\left(G,\; \tfrac{R + B}{2}\right) \quad\text{or}\quad \min(G,\; \max(R, B)) \\[4pt]
        G_{\text{new}} &= G + \alpha (G' - G), \quad \alpha \in [0, 1]
    \end{align}

    When luminance preservation is enabled (BT.709):
    \begin{equation}
        \Delta = \frac{0.7152 \cdot (G - G_{\text{new}})}{0.2126 + 0.0722}; \quad R' = \min(R + \Delta, 1); \quad B' = \min(B + \Delta, 1)
    \end{equation}

    In v0.4.5, SCNR is applied from ORIG cache with WB pre-applied: $\text{ORIG} \times \text{WB} \to \text{SCNR} \to \text{KEY}$. Idempotent.

    \subsection{Drizzle Integration}
    \label{sec:drizzle}

    \begin{equation}
        O(i, j) = \frac{\sum_p w_p \cdot K(i - x_f^p,\; j - y_f^p) \cdot I_p}{\sum_p w_p \cdot K(i - x_f^p,\; j - y_f^p)}
    \end{equation}

    Three kernels: square (top-hat), Gaussian, Lanczos-3. Flat-storage accumulator (v0.4.0): \code{Vec<f32>} + \code{Vec<u16>} replacing \code{Vec<Vec<f32>>}. Finalize uses median/MAD sigma clipping (Stetson 1987).

    \subsection{Richardson-Lucy Deconvolution}
    \label{sec:rl_deconv}

    \begin{equation}
        f^{(k+1)} = f^{(k)} \cdot \left( \frac{g}{f^{(k)} * h} * \hat{h} \right) \cdot \frac{1}{1 + \lambda}
    \end{equation}

    Tikhonov regularization as multiplicative damping ($\lambda = 0.001$ default). FFT-based convolution.

    \subsection{Background Extraction}

    \begin{equation}
        B(x, y) = \sum_{i=0}^{d} \sum_{j=0}^{d-i} a_{ij} \, x^i \, y^j
    \end{equation}

    Polynomial degree 1--5, grid 3--32, sigma-clipped sampling with \code{powi} exponentiation.

    \subsection{Wavelet Denoising}

    A trous wavelet decomposition: $I = \sum_{k=1}^{N} W_k + c_N$. Hard threshold: $W'_k = W_k$ if $|W_k| > \sigma_k t_k$, else 0. Up to 8 scales.

    \subsection{Calibration Pipeline}

    \begin{equation}
        \boxed{
            I_{\text{cal}} = \frac{I_{\text{raw}} - M_{\text{bias}} - r \cdot M_{\text{dark}}}{M_{\text{flat}} / \bar{M}_{\text{flat}}}
        }
    \end{equation}

    Masters via median combine. Flat normalized by mean. Negative pixels clamped to zero.

    \subsection{Star Detection}
    \label{sec:star_detection}

    Tile-based background (parallelized). Connected-component analysis with flux-weighted centroid:

    \begin{equation}
        \bar{x}_C = \frac{\sum_{(i,j) \in C} i \cdot (I_{ij} - b)}{\sum_{(i,j) \in C} (I_{ij} - b)}
    \end{equation}

    FWHM from second intensity moments (true \textbf{median} across detected stars, v0.4.6):

    \begin{equation}
        \text{FWHM} = 2\sqrt{2\ln 2} \cdot \sqrt{\frac{\sigma_x^2 + \sigma_y^2}{2}}
    \end{equation}

    \subsection{SPCC (Spectrophotometric Color Calibration, v0.4.2)}

    Color correction factors derived from stellar Bp-Rp color indices matched against a white reference spectrum. Optional Gaia DR3 TAP query (\code{vizier} feature flag) for real catalog data; built-in Bp-Rp estimation fallback.

    \section{FITS and ASDF I/O}
%% ============================================================================

    FITS via \crate{memmap2} with BSCALE/BZERO: $v_{\text{phys}} = \text{BSCALE} \times v_{\text{stored}} + \text{BZERO}$. Multi-BITPIX export: 16 (auto BZERO/BSCALE), $-32$ (float32), $-64$ (float64).

    ASDF: first Rust implementation. YAML tree $\to$ NdArrayMeta $\to$ binary block (zlib/bzip2/lz4) $\to$ Array2. Roman data model traversal. Auto-dispatch by extension.


%% ============================================================================
    \section{WebGPU Compute Pipeline}
%% ============================================================================

    WGSL fragment shader implementing MTF with symmetric denominator guard. \code{r32float} texture, 6-vertex fullscreen quad. Uniform buffer: $[\text{dataMin}, \text{dataMax}, \text{shadow}, \text{midtone}, \text{highlight}, W, H, \text{pad}]$ (32 bytes). CPU worker fallback with identical \code{stfTransfer} function.

%% ============================================================================
    \section{Known Limitations}
%% ============================================================================

    \begin{enumerate}[nosep]
    \item No non-linear distortion correction (barrel/pincushion)
    \item Online plate solving only (local index planned)
    \item WebGPU Chromium 113+ required; Canvas 2D fallback available
    \item No session persistence
    \item ASDF read-only (writer not implemented)
    \item No MAST API integration (manual download required)
    \item Cube ``Collapse Sum'' relabeled to ``Collapse Mean'' in v0.4.5
    \item Flatpak/Flathub distribution pending reviewer approval
    \end{enumerate}
