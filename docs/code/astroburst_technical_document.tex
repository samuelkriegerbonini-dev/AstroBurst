    \section{Executive Summary}
%% ============================================================================

    AstroBurst is a cross-platform desktop application for processing, analyzing, and visualizing astronomical FITS and ASDF images. It is the first astronomical image processor built on the \textbf{Rust/Tauri/WebGPU} stack, combining native-level performance with a modern, accessible user interface.

    The application targets two user segments:
    \begin{itemize}[nosep]
    \item Professional astronomers working with observatory data (JWST, Hubble, Roman Space Telescope, ground-based IFU instruments)
    \item Advanced astrophotographers processing deep-sky narrowband and broadband imaging data
    \end{itemize}

    Version 0.4.0 delivers a complete processing pipeline from raw FITS/ASDF ingestion through deconvolution, noise reduction, color composition, and export, with \textbf{42 backend commands} exposed through a typed TypeScript API layer.

    \subsection{What's New in v0.4.0}
    \begin{itemize}[nosep]
    \item \textbf{Star-based affine alignment} --- Triangle asterism matching with RANSAC robust estimation, supporting full affine (6-DOF) and rigid (4-DOF) transforms with automatic fallback to FFT phase correlation
    \item \textbf{Stability-based auto white balance} --- Selects the channel with lowest MAD/median coefficient of variation as reference, replacing fixed G-channel normalization
    \item \textbf{SCNR luminance redistribution} --- Green noise reduction energy is redistributed to R and B channels proportionally via ITU-R BT.709 weights
    \item \textbf{Numerical correctness audit} --- Fixed NaN ordering in median/MAD, Hann window 2$\pi$ factor, polynomial background basis, RL Tikhonov regularization, and phase correlation confidence z-score
    \item \textbf{Performance optimizations} --- Drizzle flat-storage accumulator, FFT planner hoisting, parallelized peak finding, RANSAC mask reuse, sigma-clip deviation buffer reuse
    \end{itemize}

    \subsection{What's New in v0.3.0}
    \begin{itemize}[nosep]
    \item \textbf{Richardson-Lucy deconvolution} --- FFT-based iterative deconvolution with Gaussian PSF, regularization, and deringing
    \item \textbf{Background extraction} --- Polynomial surface modeling with sigma-clipped grid sampling (subtract or divide modes)
    \item \textbf{Wavelet denoising} --- Multi-scale wavelet decomposition with per-scale sigma thresholds (up to 8 scales)
    \item \textbf{ASDF format support} --- First Rust implementation of the Advanced Scientific Data Format reader with zlib/bzip2/lz4 decompression, Roman Space Telescope data model traversal, and gWCS extraction
    \item \textbf{Redesigned tab UI} --- Bottom tabs (Info, Analysis, Headers, Export) + side panels (Processing, Compose) with resizable drag handles
    \item \textbf{Split preview contexts} --- 6 independent React contexts for minimal re-renders
    \item \textbf{Lazy-loaded panels} --- React.lazy + Suspense for fast initial load
    \end{itemize}

    \subsection{Backend (Rust)}

    The Rust backend is organized into six layers:

    \begin{description}[style=nextline, leftmargin=2cm]
    \item[\texttt{cmd/}] Tauri command handlers organized by domain (42 registered commands). Each module receives deserialized JSON arguments, delegates to core/domain logic, and returns JSON or binary responses.
    \item[\texttt{core/}] Pure computational modules: alignment (phase correlation, affine with triangle asterism matching), analysis (histogram, FFT, stars), astrometry (plate solve, WCS), compose (RGB, SCNR, white balance), cube (spectrum, frames), imaging (STF, deconvolution, wavelet, background), metadata (header discovery, filter detection), stacking (sigma-clip, drizzle).
    \item[\texttt{domain/}] Orchestration layer coordinating core modules for complex multi-step operations (calibration masters, drizzle-from-paths, cube processing).
    \item[\texttt{infra/}] Infrastructure: Arc-based zero-copy cache, config management, FITS I/O (mmap parser, MEF scanner, FITS writer), ASDF reader (YAML tree parser, binary block extractor, zlib/bzip2/lz4 decompressor, Roman data model converter, gWCS bridge), binary IPC encoding, progress tracking, PNG rendering.
    \item[\texttt{types/}] Shared type definitions: image stats, STF params, stacking/drizzle/RL configs, compose types (AlignMethod, WhiteBalance, ChannelStats), SCNR, error types, FITS header abstraction, constants.
    \item[\texttt{math/}] Mathematical primitives: NaN-safe median/MAD, sigma clipping, SIMD utilities.
    \end{description}

    All heavy computation is offloaded via \code{tokio::task::spawn\_blocking}, preventing UI freezes. Batch operations use Rayon for data-parallel execution across all available CPU cores.

    \subsection{Frontend (React + TypeScript)}

    The frontend is a single-page React application with a tab-based layout:

    \begin{description}[style=nextline, leftmargin=2cm]
    \item[\texttt{App.tsx}] Root component managing view state (empty/processing/complete), file dialog integration, drag-and-drop, resizable sidebar, and folder scanning.
    \item[\texttt{useFileQueue}] Central state machine (\code{useReducer}) tracking file status, concurrent processing queue (3 workers), resolution group detection, and auto-resampling.
    \item[\texttt{services/}] Domain-specific service modules (compose.service, fits.service, analysis.service, etc.) wrapping Tauri IPC calls with typed interfaces.
    \item[Tab System] Bottom tabs: Info (stats/header summary), Analysis (histogram, FFT, stars, spectroscopy), Headers (full header explorer), Export (FITS/PNG). Side tabs: Processing (background extraction, wavelet denoise, deconvolution), Compose (RGB with alignment method selector, drizzle, drizzle RGB).
    \item[Preview Contexts] 6 split React contexts (FileCtx, HistCtx, CubeCtx, RgbCtx, RenderCtx, RawPixelsCtx) providing scoped state to avoid cascading re-renders.
    \item[Lazy Loading] All panel components loaded via \code{React.lazy} + \code{Suspense} with spinner fallbacks.
    \end{description}

    \section{Mathematical Foundations}
    \label{sec:math}
%% ============================================================================

    \subsection{Screen Transfer Function (STF)}
    \label{sec:stf}

    The STF implements the PixInsight-style \textbf{Midtone Transfer Function} (MTF). Given a normalized pixel value $x \in [0, 1]$ and midtone parameter $m \in (0, 1)$:

    \begin{equation}
        \boxed{
            \text{MTF}(x, m) = \frac{(m - 1) \cdot x}{(2m - 1) \cdot x - m}
        }
        \label{eq:mtf}
    \end{equation}

    \textbf{Properties:}
    \begin{itemize}[nosep]
    \item $\text{MTF}(0, m) = 0$ and $\text{MTF}(1, m) = 1$ for all $m$
    \item $\text{MTF}(m, m) = 0.5$ --- the midtone maps to 50\% brightness
    \item $m < 0.5$ brightens the image; $m > 0.5$ darkens it
    \end{itemize}

    The \textbf{auto-stretch} algorithm computes the stretch parameters from image statistics. Given the pixel distribution, let $\tilde{x}$ be the median and $\text{MAD}$ the Median Absolute Deviation:

    \begin{align}
        \text{MAD} &= \text{median}\left(|x_i - \tilde{x}|\right) \label{eq:mad} \\[6pt]
        \text{shadow} &= \max\left(0,\; \tilde{x} - k \cdot \text{MAD}\right) \label{eq:shadow} \\[6pt]
        \text{midtone} &= \text{MTF}^{-1}\left(t_{\text{bg}},\; \frac{\tilde{x} - \text{shadow}}{1 - \text{shadow}}\right) \label{eq:midtone} \\[6pt]
        \text{highlight} &= 1.0 \label{eq:highlight}
    \end{align}

    where $k = 2.8$ (calibrated for deep-sky data) and $t_{\text{bg}} = 0.25$ is the target background level.

    \begin{notebox}[NaN-Safe Sorting (v0.4.0)]
        All median and MAD computations use NaN-safe comparison: NaN values are sorted to the end of the array (treated as greater than any finite value), preventing corrupted statistics that propagate through sigma clipping, auto-STF, and white balance.
    \end{notebox}

    \subsection{Asinh Stretch}

    The \textbf{arcsinh transfer function} provides astronomically-correct normalization that preserves flux ratios between stars of different magnitudes:

    \begin{equation}
        \boxed{
            f(x) = \frac{\operatorname{asinh}(\alpha \cdot x)}{\operatorname{asinh}(\alpha)}
        }
        \label{eq:asinh}
    \end{equation}

    where $\alpha$ controls the stretch aggressiveness. This function maps $[0, 1] \to [0, 1]$ and is approximately linear for small $x$ (preserving faint detail) while compressing bright values logarithmically.

    \subsection{Cross-Correlation Alignment (Phase Correlation)}
    \label{sec:alignment_phase}

    Frame registration uses \textbf{phase correlation} in the frequency domain. Given reference frame $R$ and target frame $T$:

    \begin{equation}
        \text{shift} = \arg\max_{(\Delta x, \Delta y)} \mathcal{F}^{-1}\!\left\{\frac{\hat{R} \cdot \hat{T}^*}{|\hat{R} \cdot \hat{T}^*|}\right\}
        \label{eq:phase_corr}
    \end{equation}

    where $\hat{R} = \mathcal{F}\{R\}$ denotes the 2D DFT applied with a \textbf{Hann window} $w(n) = 0.5(1 - \cos(2\pi n / N))$. The normalized cross-power spectrum produces a sharp peak at the integer translation offset. Sub-pixel refinement is achieved via parabolic interpolation on the correlation surface, followed by bicubic Catmull-Rom resampling for the actual shift.

    Confidence is measured as the z-score of the peak:
    \begin{equation}
        \text{confidence} = \frac{P_{\text{peak}} - \mu_P}{\sigma_P}
        \label{eq:confidence}
    \end{equation}

    where $\mu_P$ and $\sigma_P$ are the mean and standard deviation of the correlation surface. A coarse-to-fine strategy downsamples to 512$\times$512 for initial offset estimation, then refines on a 512$\times$512 crop centered on the coarse estimate.

    \textbf{Corrects:} pure translation (sub-pixel). \textbf{Does not correct:} rotation, scale.

    \subsection{Star-Based Affine Alignment (v0.4.0)}
    \label{sec:alignment_affine}

    For images with rotational misalignment, AstroBurst provides a star-based alignment pipeline using triangle asterism matching and RANSAC robust estimation.

    \subsubsection{Star Detection and Triangle Construction}

    Stars are detected via sigma-clipped background estimation and connected-component analysis (see Section~\ref{sec:star_detection}). The top 80 stars by flux are selected from each image. For every triplet of stars $(s_i, s_j, s_k)$ with minimum side length $> 20$\,px, a triangle descriptor is computed:

    \begin{equation}
        T = \left(\frac{d_{\text{mid}}}{d_{\text{short}}},\; \frac{d_{\text{long}}}{d_{\text{short}}}\right)
        \label{eq:triangle_desc}
    \end{equation}

    where $d_{\text{short}} \leq d_{\text{mid}} \leq d_{\text{long}}$ are the sorted side lengths. These ratios are \textbf{invariant} under translation, rotation, and uniform scaling.

    \subsubsection{Triangle Matching}

    For each pair of reference triangle $T_r$ and target triangle $T_t$, a match is declared if:

    \begin{equation}
        \left|\frac{d_{\text{mid}}^r}{d_{\text{short}}^r} - \frac{d_{\text{mid}}^t}{d_{\text{short}}^t}\right| < \epsilon
        \quad \text{and} \quad
        \left|\frac{d_{\text{long}}^r}{d_{\text{short}}^r} - \frac{d_{\text{long}}^t}{d_{\text{short}}^t}\right| < \epsilon
        \label{eq:triangle_match}
    \end{equation}

    with $\epsilon = 0.005$. Matching triangles vote for star correspondences; pairs with $\geq 3$ votes are accepted.

    \subsubsection{RANSAC Transform Estimation}

    Given $n$ star correspondences $\{(\mathbf{p}_i^r, \mathbf{p}_i^t)\}_{i=1}^n$, the affine transform is estimated via RANSAC (500 iterations):

    \begin{equation}
        \begin{pmatrix} x_t \\ y_t \end{pmatrix} =
        \begin{pmatrix} a & b \\ c & d \end{pmatrix}
        \begin{pmatrix} x_r \\ y_r \end{pmatrix} +
        \begin{pmatrix} t_x \\ t_y \end{pmatrix}
        \label{eq:affine}
    \end{equation}

    The 6 parameters $(a, b, c, d, t_x, t_y)$ are solved via least squares on a 3-point sample (affine) or 2-point sample (rigid, where $a = \cos\theta$, $b = -\sin\theta$, $c = \sin\theta$, $d = \cos\theta$). Inliers are points with reprojection error $< 2$\,px.

    \textbf{Sanity checks} reject transforms with: offset $> 25\%$ of image dimension, rotation $> 5^\circ$, scale outside $[0.85, 1.15]$, residual $> 3$\,px, or inlier ratio $< 30\%$.

    \textbf{Fallback chain:} affine (8+ matches) $\to$ rigid (5+ matches) $\to$ phase correlation $\to$ identity.

    \subsubsection{Image Warping}

    The aligned image is produced by inverse-mapping each output pixel through the affine transform and sampling the source image via bicubic Catmull-Rom interpolation:

    \begin{equation}
        I_{\text{out}}(x, y) = \text{bicubic}\!\left(I_{\text{in}},\; \mat{A}^{-1} \begin{pmatrix} x \\ y \end{pmatrix}\right)
    \end{equation}

    Pixels mapping outside the source bounds are set to NaN. Warping is parallelized across rows via Rayon.

    \subsection{Sigma-Clipped Stacking}
    \label{sec:sigma_stack}

    \begin{algorithm}[H]
\SetAlgoLined
\SetKwInOut{Input}{Input}
\SetKwInOut{Output}{Output}
\Input{Aligned frames $\{F_1, F_2, \ldots, F_N\}$, thresholds $\sigma_{\text{low}}, \sigma_{\text{high}}$, max iterations $K$}
\Output{Stacked image $S$}
\ForEach{pixel position $(i, j)$}{
    $V \leftarrow \{F_1(i,j), F_2(i,j), \ldots, F_N(i,j)\}$\;
    \For{$k = 1$ \KwTo $K$}{
        $\mu \leftarrow \text{mean}(V)$\;
        $\sigma \leftarrow \text{std}(V)$\;
        $V' \leftarrow \{v \in V : \mu - \sigma_{\text{low}} \cdot \sigma \leq v \leq \mu + \sigma_{\text{high}} \cdot \sigma\}$\;
        \If{$|V'| = |V|$ \textbf{or} $|V'| < 3$}{
            \textbf{break}\;
        }
        $V \leftarrow V'$\;
    }
    $S(i,j) \leftarrow \text{mean}(V)$\;
}
\caption{Iterative Sigma-Clipped Stacking}
\label{alg:sigma_clip}
    \end{algorithm}

    Default parameters: $\sigma_{\text{low}} = \sigma_{\text{high}} = 3.0$, $K = 5$. The deviation buffer is allocated once and reused per pixel to avoid heap allocation overhead.

    \subsection{Drizzle Integration}
    \label{sec:drizzle}

    Drizzle (Variable-Pixel Linear Reconstruction) maps input pixels onto a higher-resolution output grid:

    \begin{equation}
        O(i, j) = \frac{\sum_p w_p \cdot K(i - x_f^p,\; j - y_f^p) \cdot I_p}{\sum_p w_p \cdot K(i - x_f^p,\; j - y_f^p)}
        \label{eq:drizzle}
    \end{equation}

    Three kernel options:

    \begin{align}
        K_{\text{square}}(\Delta x, \Delta y) &=
        \begin{cases}
            1 & \text{if } |\Delta x| \leq \frac{d}{2} \text{ and } |\Delta y| \leq \frac{d}{2} \\
            0 & \text{otherwise}
        \end{cases}
        \label{eq:kernel_square} \\[6pt]
        K_{\text{gaussian}}(\Delta x, \Delta y) &= \exp\!\left(-\frac{\Delta x^2 + \Delta y^2}{2\sigma_K^2}\right)
        \label{eq:kernel_gaussian} \\[6pt]
        K_{\text{lanczos3}}(\Delta x, \Delta y) &= L_3(\Delta x) \cdot L_3(\Delta y)
        \label{eq:kernel_lanczos}
    \end{align}

    \begin{notebox}[Drizzle Accumulator (v0.4.0)]
        The drizzle accumulator uses a flat \code{Vec<f32>} + \code{Vec<u16>} count array instead of \code{Vec<Vec<f32>>}, eliminating millions of heap allocations for typical 4K+ output images. Per-pixel storage is pre-allocated as $2 \times N_{\text{frames}}$ slots with contiguous memory layout.
    \end{notebox}

    \subsection{White Balance (v0.4.0)}
    \label{sec:white_balance}

    Auto white balance selects the most stable channel as reference using the coefficient of variation (MAD/median ratio):

    \begin{equation}
        k^* = \arg\min_{k \in \{R, G, B\}} \frac{\text{MAD}_k}{\tilde{C}_k}
        \label{eq:wb_stability}
    \end{equation}

    The reference channel $k^*$ receives weight 1.0; other channels are scaled by the ratio of medians:

    \begin{equation}
        w_j = \frac{\tilde{C}_{k^*}}{\tilde{C}_j}, \quad j \neq k^*; \qquad w_{k^*} = 1
        \label{eq:wb_weights}
    \end{equation}

    This replaces the v0.3 fixed G-channel reference, which could amplify noise in narrowband data where the G channel (typically OIII) may have the highest noise.

    \subsection{SCNR (Subtractive Chromatic Noise Reduction) (v0.4.0)}
    \label{sec:scnr}

    SCNR removes green channel excess with optional luminance preservation. Two limit methods:

    \begin{align}
        G'_{\text{average}} &= \min\!\left(G,\; \frac{R + B}{2}\right) \label{eq:scnr_avg} \\[4pt]
        G'_{\text{max}} &= \min\!\left(G,\; \max(R, B)\right) \label{eq:scnr_max}
    \end{align}

    The corrected green value incorporates the amount parameter:

    \begin{equation}
        G_{\text{new}} = G + \alpha \cdot (G' - G), \quad \alpha \in [0, 1]
    \end{equation}

    When luminance preservation is enabled, the luminance lost from green reduction is redistributed to R and B using ITU-R BT.709 weights:

    \begin{align}
        L_{\text{lost}} &= 0.7152 \cdot (G - G_{\text{new}}) \\[4pt]
        \Delta &= \frac{L_{\text{lost}}}{0.2126 + 0.0722} \\[4pt]
        R' &= \min(R + \Delta,\; 1.0) \\[4pt]
        B' &= \min(B + \Delta,\; 1.0)
    \end{align}

    This replaces the v0.3 approach which attempted to add lost luminance back into G itself (circular adjustment).

    \subsection{Richardson-Lucy Deconvolution}
    \label{sec:rl_deconv}

    The Richardson-Lucy algorithm iteratively estimates the true image $f$ given observed image $g$ and point spread function $h$:

    \begin{equation}
        f^{(k+1)} = f^{(k)} \cdot \left( \frac{g}{f^{(k)} * h} * \hat{h} \right)
        \label{eq:rl}
    \end{equation}

    Convolutions are computed in the frequency domain via FFT. Tikhonov regularization is applied as multiplicative damping in the update step (v0.4.0 fix):

    \begin{equation}
        f^{(k+1)}_{\text{reg}} = f^{(k+1)} \cdot \frac{1}{1 + \lambda}
    \end{equation}

    where $\lambda$ is the regularization parameter (default 0.001). This is numerically stable and matches the standard RL formulation, replacing the v0.3 approach of dividing by $1 + \lambda \nabla^2 f$.

    \subsection{Background Extraction}
    \label{sec:background}

    Background is modeled as a polynomial surface fitted to sigma-clipped grid samples:

    \begin{equation}
        B(x, y) = \sum_{i=0}^{d} \sum_{j=0}^{d-i} a_{ij} \, x^i \, y^j
        \label{eq:background}
    \end{equation}

    The polynomial basis functions use explicit \code{powi} exponentiation (v0.4.0 fix), replacing the iterative division approach that accumulated numerical drift at degree $\geq 3$.

    \subsection{Wavelet Denoising}
    \label{sec:wavelet}

    Multi-scale wavelet denoising uses the \`{a} trous algorithm. The image is decomposed into $N$ wavelet scales $W_k$ and a residual $c_N$:

    \begin{equation}
        I = \sum_{k=1}^{N} W_k + c_N
    \end{equation}

    Denoising applies a hard threshold to each scale:

    \begin{equation}
        W'_k(x, y) = \begin{cases}
                         W_k(x, y) & \text{if } |W_k(x, y)| > \sigma_k \cdot t_k \\
                         0 & \text{otherwise}
        \end{cases}
    \end{equation}

    Up to 8 decomposition scales are supported.

    \subsection{Calibration Pipeline}
    \label{sec:calibration}

    \begin{equation}
        \boxed{
            I_{\text{calibrated}} = \frac{I_{\text{raw}} - M_{\text{bias}} - r \cdot M_{\text{dark}}}{M_{\text{flat}} / \tilde{M}_{\text{flat}}}
        }
        \label{eq:calibration}
    \end{equation}

    where $\tilde{M}_{\text{flat}}$ is the \textbf{median} of the flat field (v0.4.0 fix, replacing mean), preventing scale-dependent artifacts when flat values are far from 1.0.

    \subsection{World Coordinate System (WCS)}
    \label{sec:wcs}

    WCS transforms use the \textbf{CD matrix formulation} with TAN (gnomonic) projection:

    \begin{equation}
        \begin{pmatrix} u \\ v \end{pmatrix} =
        \begin{pmatrix} \text{CD}_{1,1} & \text{CD}_{1,2} \\ \text{CD}_{2,1} & \text{CD}_{2,2} \end{pmatrix}
        \begin{pmatrix} x - \text{CRPIX}_1 \\ y - \text{CRPIX}_2 \end{pmatrix}
        \label{eq:wcs_forward}
    \end{equation}

    \subsection{FFT Power Spectrum}
    \label{sec:fft}

    \begin{equation}
        P(u, v) = \log_{10}\!\left(1 + |\hat{I}(u, v)|^2\right)
        \label{eq:power_spectrum}
    \end{equation}

    The Hann window is correctly applied as $w(n) = 0.5(1 - \cos(2\pi n / N))$ (v0.4.0 fix; v0.3 used $\sin(\pi n / N)$ which produces a triangular window).

    \subsection{Star Detection (PSF Centroid)}
    \label{sec:star_detection}

    Background is estimated via iterative sigma-clipped median. For each connected component $C$:

    \begin{align}
        \bar{x}_C &= \frac{\sum_{(i,j) \in C} i \cdot (I_{ij} - b)}{\sum_{(i,j) \in C} (I_{ij} - b)}
        & \bar{y}_C &= \frac{\sum_{(i,j) \in C} j \cdot (I_{ij} - b)}{\sum_{(i,j) \in C} (I_{ij} - b)} \label{eq:centroid} \\[6pt]
        F_C &= \sum_{(i,j) \in C} (I_{ij} - b)
        & \text{SNR}_C &= \frac{F_C}{\sqrt{|C| \cdot \sigma_b^2}} \label{eq:flux_snr}
    \end{align}

    FWHM from second intensity moments:

    \begin{equation}
        \text{FWHM} = 2\sqrt{2\ln 2} \cdot \sqrt{\frac{\sigma_x^2 + \sigma_y^2}{2}}
        \label{eq:fwhm}
    \end{equation}

    \section{FITS I/O Implementation}
    \label{sec:fits_io}
%% ============================================================================

    FITS files are opened via \crate{memmap2}, avoiding loading entire files into memory. For integer data (BITPIX $> 0$), the BSCALE/BZERO transform is applied:

    \begin{equation}
        \text{value}_{\text{physical}} = \text{BSCALE} \times \text{value}_{\text{stored}} + \text{BZERO}
        \label{eq:bscale}
    \end{equation}

%% ============================================================================
    \section{ASDF I/O Implementation}
    \label{sec:asdf_io}
%% ============================================================================

    AstroBurst includes the first Rust implementation of the \textbf{Advanced Scientific Data Format} (ASDF) reader, targeting NASA's Roman Space Telescope data products. Binary blocks may be compressed with zlib (\crate{flate2}), bzip2 (\crate{bzip2}), or lz4 (\crate{lz4\_flex}). Auto-dispatch routes \code{.asdf} files transparently through the same command layer as \code{.fits}.

    \section{WebGPU Compute Pipeline}
    \label{sec:webgpu}
%% ============================================================================

    The STF stretch is implemented as a WebGPU compute shader (WGSL) for real-time rendering with workgroup size 256. Falls back to Canvas 2D \code{putImageData} when WebGPU is unavailable.

    \section{Security Considerations}
    \label{sec:security}
%% ============================================================================

    \begin{itemize}[nosep]
    \item API keys stored in OS-native app data directory, never transmitted beyond the configured API endpoint.
    \item File system access constrained by Tauri capabilities. Permissions scoped to home, desktop, and appdata directories.
    \item No telemetry, analytics, or network calls except for explicit plate solving requests.
    \item FITS and ASDF files processed locally; no data leaves the machine unless plate solving is triggered.
    \end{itemize}

%% ============================================================================
    \section{Known Limitations}
    \label{sec:limitations}
%% ============================================================================

    \begin{enumerate}[nosep]
    \item \textbf{Unwired commands} --- Plate solving, calibration, sigma-clip stacking, tile deep zoom, WCS overlay, narrowband auto-detection, and config UI panels are backend-ready but lack full frontend integration
    \item \textbf{Affine alignment limits} --- Requires detectable point sources; fails gracefully (falls back to phase correlation) on pure nebula fields without stars
    \item \textbf{No non-linear distortion correction} --- Affine model handles translation, rotation, and scale but not barrel/pincushion or field-variable distortion
    \item \textbf{Online plate solving only} --- Local index-based solving is planned
    \item \textbf{WebGPU requirement} --- Chromium 113+ needed; Canvas 2D fallback available
    \item \textbf{No session persistence} --- Reopening the app starts a fresh workspace
    \item \textbf{ASDF read-only} --- Writer not yet implemented
    \item \textbf{No MAST API integration} --- JWST/Roman data must be downloaded manually
    \end{enumerate}
